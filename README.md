## 🚀 茂亨外贸版Odoo (Trade Odoo 17) 一键部署脚本 - 零基础教程

### **核心特色**
- **为外贸企业优化**：专为外贸业务流程（客户关系、订单跟进、采购管理、库存物流、财务对账等）深度定制。
- **一键智能部署**：只需几个简单命令，即可在您的VPS（云服务器）上部署一套完整的、开箱即用的Odoo系统。
- **无需懂代码**：本指南所有步骤均以代码块形式提供，您只需在服务器上**复制、粘贴、回车**即可。
- **包含中文支持**：已内置中文字体，解决PDF中文显示和打印问题。
- **安全与优化**：自动配置防火墙、SSL证书（如果使用域名）、数据库备份和系统性能优化。

### **第一步：连接您的VPS（云服务器）**

1.  从您的服务器供应商（如阿里云、腾讯云、DigitalOcean等）获取服务器的**IP地址**、**用户名**（通常是 `root`）和**密码**（或SSH密钥）。
2.  打开您电脑上的**终端**（Windows可使用 PowerShell 或 PuTTY，Mac/Linux 直接打开“终端”应用）。
3.  使用SSH命令连接服务器（将 `您的IP` 替换为真实IP）：
    ```bash
    ssh root@您的IP
    ```
4.  输入密码（输入时不会显示字符），成功登录后，您会看到类似 `root@server:~#` 的命令提示符。

---

### **第二步：复制并运行一键部署脚本**

#### **1. 下载脚本**
登录服务器后，直接执行以下**一条命令**来下载部署脚本：
```bash
wget https://gist.githubusercontent.com/ai-assistant-example/raw/main/odoo-manager.sh
```
> **注意**：以上为示例链接，实际操作时请替换为您或脚本维护者提供的**最新、有效**的脚本直链地址。如果此链接不可用，您可能需要从可靠的来源（如官方网站、技术社区）获取最新的脚本文件。

#### **2. 赋予脚本执行权限**
```bash
chmod +x odoo-manager.sh
```

#### **3. 初始化系统环境 (必须执行一次)**
这个命令会为服务器安装所有必需的软件（如Docker、Nginx、字体等）。
```bash
sudo ./odoo-manager.sh init
```
这个过程根据网络情况可能需要5-15分钟，请耐心等待直到出现 **“环境初始化完成”** 的提示。

#### **4. 一键部署您的Odoo系统（最快方式）**
这是最简单的部署方式，适合想快速开始使用的用户。只需将 `mycompany-odoo` 换成您喜欢的名字（如 `yourcompany-odoo`）。
```bash
sudo ./odoo-manager.sh deploy mycompany-odoo
```
脚本将自动完成所有配置，并在几分钟后为您提供一个可访问的Odoo系统。

**部署成功后，您会看到类似下面的信息：**

  直接访问地址： http://[您的服务器IP]:8069

### **🎯 新手常见问题 (Q&A)**

**Q1: 部署完成后，我如何登录系统？**
A1: 在浏览器地址栏输入部署成功时显示的地址，通常是 `http://您的服务器IP:8069`。使用管理员账号（默认 `admin`）和自动生成的密码（在 `/opt/yourname-odoo/.env` 文件中查找 `ADMIN_PASSWORD` 一行）登录。

**Q2: 如何找到自动生成的密码？**
A2: 在服务器上执行这条命令（将 `yourname-odoo` 替换为您的实例名）：
```bash
cat /opt/yourname-odoo/.env | grep ADMIN_PASSWORD
```
屏幕上显示的一长串字符就是您的初始密码。

**Q3: 我想用我自己的域名访问（如 `erp.mycompany.com`），怎么办？**
A3: 在**第三步**中不要用 `deploy mycompany-odoo`，而是直接运行 `sudo ./odoo-manager.sh deploy`，然后根据交互式菜单选择 **“域名模式”**，之后按提示输入您的域名即可。脚本会引导您完成SSL证书申请。

**Q4: 数据会自动备份吗？**
A4: **是的**。脚本已设置自动任务，每天凌晨2点备份数据库，您可以在实例目录下的 `backups/` 文件夹找到备份文件。

**Q5: 部署失败或遇到错误怎么办？**
A5:
1.  **检查网络**：确保您的VPS可以正常访问互联网。
2.  **查看日志**：运行 `sudo ./odoo-manager.sh logs` 可以查看详细错误信息。
3.  **重新初始化**：如果环境问题，可尝试再次运行 `sudo ./odoo-manager.sh init`。
4.  **寻求帮助**：将错误日志截图，联系您的服务器提供商或茂亨的技术支持人员。
